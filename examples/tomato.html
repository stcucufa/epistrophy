Flipbook animation example (tomato)

LOADING...

A simple flipbook animation loop is at its core a loop showing an image for a small interval (e.g., at a modest 9 frames per second, the interval is 1000 / 9, or around 111, milliseconds). The images are loaded first with map, and the loop runs through all frames using each (and is nested inside another repeat to run indefinitely).

Different animation loops can be chosen by listening to buttons and choosing a corresponding animation loop (which map enables by naming fibers by the keys of the object that it maps over). In order to make transitions smoother, moving to the next loop only happens after a whole sequence has played, which is achieved by wrapping the each instruction into either, enabling it to run even when the fiber is cancelled.
